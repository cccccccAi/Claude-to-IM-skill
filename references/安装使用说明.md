# Claude-to-IM 安装使用说明

> 将 Claude Code 接入飞书、Telegram、Discord 或 QQ，手机直接发消息就能用 Claude 写代码。

---

## 一、前提条件

- **Node.js >= 20**（`node --version` 确认）
- **Claude Code CLI** 已安装并登录（`claude --version` 能正常输出）

---

## 二、安装

```bash
npx skills add cccccccAi/Claude-to-IM-skill
```

出现平台选择列表后，选 `Claude Code (.claude/skills)` 回车确认。

---

## 三、初始化配置

在 Claude Code 里运行：

```
/claude-to-im setup
```

向导会依次询问：

1. **选择渠道**：飞书 / Telegram / Discord / QQ（可多选）
2. **填写凭证**：各平台的 Bot Token / App ID 等
3. **通用设置**：
   - Runtime：推荐 `cli-print`（无需 API Key，用 OAuth 订阅）
   - 工作目录：Claude 执行任务的默认目录
   - API Base URL（可选）：使用第三方代理时填入，例如 `https://your-proxy.com/v1`
   - Model（可选）：留空使用默认
   - Mode：`code`（默认）/ `plan` / `ask`

配置完成后自动验证 Token 并写入 `~/.claude-to-im/config.env`。

---

## 四、各平台凭证获取

### 飞书

1. 进入 [飞书开放平台](https://open.feishu.cn/app) → 创建自建应用
2. **凭证与基本信息** 页面取 App ID 和 App Secret
3. **权限管理** → 批量配置，粘贴下方 JSON：

```json
{
  "scopes": {
    "tenant": [
      "im:message",
      "im:message:send_as_bot",
      "im:message:readonly",
      "im:message.p2p_msg:readonly",
      "im:resource",
      "im:chat.access_event.bot_p2p_chat:read",
      "im:chat.members:bot_access",
      "contact:user.employee_id:readonly",
      "application:application:self_manage"
    ],
    "user": ["im:chat.access_event.bot_p2p_chat:read"]
  }
}
```

4. **添加应用能力** → 启用机器人
5. **事件与回调** → 事件订阅方式选 **长连接** → 添加 `im.message.receive_v1` 事件
6. **版本管理与发布** → 创建版本 → 提交审核 → 管理员后台审批通过

> ⚠️ 版本未发布前机器人不会响应消息

---

### Telegram

1. 找 `@BotFather` → `/newbot` → 按提示创建 → 复制 Token（格式：`数字:字符串`）
2. 获取 Chat ID：给机器人发一条消息，浏览器打开
   `https://api.telegram.org/bot<TOKEN>/getUpdates`，找 `"chat":{"id":...}` 的数字
3. 可选：找 `@userinfobot` 获取自己的 User ID 用于授权白名单

---

### Discord

1. 进入 [Discord Developer Portal](https://discord.com/developers/applications) → 新建应用
2. Bot 页签 → Reset Token → 复制
3. 开启 **Message Content Intent**（Privileged Gateway Intents 下）
4. OAuth2 → URL Generator → 勾选 `bot` + 权限（Send Messages / Read History / View Channels）→ 邀请机器人进服务器
5. 右键用户 → Copy User ID（需先在设置开启 Developer Mode）

---

### QQ

1. 进入 [QQ Bot 开放爪](https://q.qq.com/qqbot/openclaw)
2. 创建机器人，获取 App ID 和 App Secret
3. 扫码将机器人添加为好友（沙箱测试）
4. Allowed Users 填 `user_openid`（非 QQ 号），首次可留空

> ⚠️ QQ 目前仅支持私聊，不支持群聊/频道，无流式输出，无权限按钮

---

## 五、启动与常用命令

```
/claude-to-im start       # 启动后台服务
/claude-to-im status      # 查看运行状态
/claude-to-im logs        # 查看最近日志（默认 50 行）
/claude-to-im logs 200    # 查看最近 200 行日志
/claude-to-im stop        # 停止服务
/claude-to-im doctor      # 诊断问题
/claude-to-im reconfigure # 修改配置
```

---

## 六、进阶配置

手动编辑 `~/.claude-to-im/config.env` 可配置更多选项：

```bash
# 运行模式
CTI_RUNTIME=cli-print

# 危险模式：跳过所有工具授权提示（仅限 cli-print，个人机器推荐开启）
CTI_DANGEROUSLY_SKIP_PERMISSIONS=true

# 第三方 API 代理
ANTHROPIC_API_KEY=<YOUR_KEY>
ANTHROPIC_BASE_URL=https://your-proxy.com/v1

# 代理（cli-print 连接 Anthropic 需要）
HTTPS_PROXY=http://your-proxy:port
```

修改后执行 `/claude-to-im stop` → `/claude-to-im start` 生效。

## 七、常见问题

| 问题         | 解决                                     |
| ---------- | -------------------------------------- |
| 服务启动失败     | 运行 `/claude-to-im doctor` 查看诊断结果       |
| 收不到消息      | 检查 Token 是否正确，飞书需确认版本已发布               |
| 请求超时（3分钟）  | 任务太复杂，Claude 会自动返回超时提示                 |
| CLI 未登录报错  | 在终端执行 `claude auth login` 重新登录         |
| API Key 错误 | 检查 `config.env` 中的 `ANTHROPIC_API_KEY` |
